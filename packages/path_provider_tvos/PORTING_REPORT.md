# path_provider_tvos — maintained native implementation

`path_provider_foundation` (the iOS/macOS implementation) is a
dart:ffi / native-assets plugin, which the flutter-tvos toolchain
cannot build for tvOS. The porter therefore only emitted a skeleton.
This package is now a **hand-written, maintained tvOS implementation**
— not generated output.

## Status

✅ Working native implementation.

- `tvos/Classes/PathProviderPlugin.swift` answers the
  `plugins.flutter.io/path_provider` channel using `NSTemporaryDirectory()`
  and `NSSearchPathForDirectoriesInDomains` (documents / library /
  caches / application-support), creating the support & cache
  directories on first use — mirrors `path_provider_foundation`,
  Foundation-only, tvOS-safe.
- `lib/path_provider_tvos.dart` is a thin federated forwarder over the
  same method-channel contract.

## tvOS behaviour

| API | tvOS |
|---|---|
| `getTemporaryPath` | ✅ `NSTemporaryDirectory()` |
| `getApplicationDocumentsPath` | ✅ `…/Documents` |
| `getApplicationSupportPath` | ✅ `…/Library/Application Support` (auto-created) |
| `getApplicationCachePath` | ✅ `…/Library/Caches` (auto-created) |
| `getLibraryPath` | ✅ `…/Library` |
| `getDownloadsPath` | ❌ returns `null` — no user Downloads dir on tvOS |
| `getExternalStoragePath` / `*CachePaths` / `*StoragePaths` | ❌ Android-only — `UnsupportedError` (same as iOS/macOS) |
